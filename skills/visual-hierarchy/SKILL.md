# visual-hierarchy

## Purpose
Tune visual priority so users can identify context, key information, and next
actions quickly.

## Auto Activation
Use when screens feel cluttered, flat, hard to scan, overly decorative, or when
editing headings, density, spacing, grouping, status emphasis, or page order.

## Do Not Activate
Do not use for purely functional backend work or copy-only changes where layout
priority is unaffected.

## Detect
Look for competing headings, oversized text, weak grouping, too many cards,
unclear primary actions, low contrast, dense dashboards, and misordered content.

## Responsibilities
- Put task context, primary action, and critical status in predictable places.
- Use spacing, alignment, weight, and color for priority rather than decoration.
- Reduce competing emphasis.
- Match type scale to the surface size and density.
- Make repeated content easy to compare.

## Never Do
- Use hero-scale typography inside compact product surfaces.
- Emphasize everything at once.
- Add decorative contrast that does not clarify priority.
- Reorder business information without understanding the workflow.

## Cooperates With
ui-designer, anti-ai-ui, internal-business-ui, dashboard-design, table-design,
form-design, copywriting-ui.

## Final Checklist
- The user's first read is intentional.
- Primary and secondary actions are visually distinct.
- Grouping supports the workflow.
- Text scale matches the container.
- Color and emphasis clarify meaning.

## Examples
- Reduce a settings page from equal-weight sections to clear groups with one
  primary save action.
- Rebalance a dashboard so alerts and overdue work outrank secondary metrics.
