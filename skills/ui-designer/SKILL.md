# ui-designer

## Purpose
Create practical product UI that is clear, usable, and visually restrained.

## Auto Activation
Use when creating or changing screens, components, forms, tables, dashboards, navigation, toolbars, or visual states.

## Do Not Activate
Do not use for backend-only work or cosmetic changes explicitly outside scope.

## Detect
Look for UI components, design systems, CSS tokens, forms, tables, empty states, modals, responsive layouts, icons, and user workflows.

## Responsibilities
- Prioritize workflow clarity and scanning.
- Use predictable controls for common actions.
- Keep hierarchy, alignment, spacing, and contrast professional.
- Design real empty, loading, error, disabled, and success states.
- Match the product domain and existing style.

## Never Do
- Build a marketing landing page when the task needs a usable app screen.
- Hide core controls behind decorative composition.
- Use oversized hero typography inside compact product surfaces.
- Let visual polish reduce usability.

## Cooperates With
anti-ai-ui, html-css, javascript, ux, accessibility, testing, code-review.

## Final Checklist
- The main workflow is immediately usable.
- Controls match user expectations.
- States are present and clear.
- Layout is responsive without overlap.
- Visual style fits the product, not a template.

## Examples
- Turn a form into aligned field groups with inline validation and a clear submit state.
- Improve a data table with readable density, filters, sorting affordances, and empty state.
